Forms

Forms are how we manage data we are going to send to the server. The entire form content is wrapped in a <form> tag. Forms have attributes that describe how they should interact with the server - the most important of these are method (which describes the HTTP method to use) and action (which describes where the data should be sent). See MDN's form reference for a complete list and more details

The form's data are stored in <input> tags to allow for user interaction. There are many types of <input> tags and related attributes, see MDN's reference on inputs for a comprehensive list of input types and other related input attributes.

<input> tags are labeled by <label> tags. You can use the for attribute on your <label> tag to reference the labeled input element's id.

Why do we need programming languages?

  • branching/condtionals: if a certain state is observed, do a thing
  • loops: doing the same thing to a whole list of things
  • input/output: consuming data and doing something with it

Variables

In programming languages, information is stored in things called variables.

in Javascript variables are created using the keyword var. For example: var x = 1;

Variables can store different types of data, for example:

  • Numbers: sometimes called integers (without a decimal) and floats (with a decimal). In Javascript we just call them all numbers
  • Strings: some text, programming languages like Javascript are good at manipulating text

Operators

In the above example (var x = 1;), we call the equals sign (=) the "operator". The operator = assigns a value to a variable. The above example assigns the value 1 to the variable x.

There are other operators used to manipulate values, for example: +, -, *, etc.

Functions

Functions fundamentally have three parts:

  • Take in some information. We call this input information 'arguments'. You can supply a function with as many arguments as you like.
  • Do something, probably using the information that has been 'passed in' as arguments.
  • Return something. Functions in Javascript always return something. If they don't return a specific value, they return a value called undefined.

For example:

function sum(a, b) {
  var result = a + b;
  return result;
}

Javascript Language Rules

  • Lines end with a semicolon (;)
  • Some words are reserved, for example var.
  • Variable names can't start with numbers, but they can contain numbers.
  • Variable names should be camel case for example: forExample

CSS selectors

CSS selectors are how you get things on the page to style them in your CSS. There are a few basic ways to do this:

  • by element: a {color: red;}
  • by class: .danger {color: red;}
  • by id: #warning-message {color: red;}

When it comes to ids and classes, remember:

  • An id should only appear once on the page. Ids should be unique and elements can only have one id.
  • Classes may be applied to many elements on the page.
  • Elements can have many classes!

Select on multiple things on a single element by chaining selectors together without a space, for example: a.danger.btn {color: red;}

Descendant CSS selectors

You can select nested elements by putting a space between selectors. For example: header a selects all the <a> tags in the header.

Some rules of thumb when nesting CSS selectors:

  • Keep it shallow! Deeply nested selectors can get confusing.
  • Omit any unnecessary selectors. Keep it as simple as possible!

Pseudo-selectors!

Pseudo-selectors allow you to create CSS selectors that select elements based on a particular state of that element (called pseudo-classes). They are denoted by a colon. For example: a:hover selects the <a> tags that are currently being hovered over by the cursor.

A list of all the available pseudo-classes is available on MDN.

Some rules of thumb when using pseudo-selectors:

  • Use them sparingly! If you're uncertain, you probably don't need it.
  • Don't get too clever -- It's really easy for pseudo-selectors to get confusing!

CSS Layouts

The way we've managed CSS layouts has evolved over the years. In chronological order:

  • Tables: using the <table> tag, you can create columns and rows. This (sort of) allows you to manage complex layouts, but it's really clunky and has some limitations
  • Floats: using the CSS property float:left; or float:right; pulls elements out of the layout flow. This has been used to create column layouts, but floating has some idiosyncrasies that require proactive management. Floated elements must be "cleared", and this can cause problems.
  • Flexbox: using the CSS property display:flex; is the modern way to manage complex layouts with CSS. It allows you to flexibly size child elements using some special CSS properties such as flex. You can find more info at MDN.

There are 3 ways to set child width when using display:flex;:

  • By default the element will be the width of the content inside it
  • Horizontal width in px using the width property
  • Proportional sizing using the flex property. Elements with a flex value will get a proportion of the entire remaining width of the parent element based on the ratio of it's flex value to those of all other child elements with a flex property

Things on a web page outside the document itself...

There are a bunch of things that make up a web page aside from the actual document content itself. Stuff like...

  • Window/Tab/Page title
  • Description
  • Metadata about the page (used by other pages, like Facebook and Google)
  • Javascript files
  • CSS files

Generally speaking, metadata goes in the <head> tag, and page content goes in the <body> tag.

Ways to style your HTML

  • 'Inline styling' with the style attribute. In most cases, this is the wrong way to style your web page, but can be handy for testing things out.
    example: <footer style="text-align:center;"></footer>
  • Embedding styling rules in your HTML page in a <style> tag. This can work for small pages, but is usually not preferred.
    example: <style>footer { text-align:center; }</style>
  • Linking to a CSS file that contains your styling rules using a <link> tag. This is usually the best thing to do as it separates your style completely from your markup.
    example: <link rel="stylesheet" href="styles.css">

IMPORTANT NOTE! Inline styles take precedence over styling rules. Still, you probably shouldn't use them.

Positioning Elements with CSS

When you use anything other that "static" (the default value for position), you'll be able to use the properties "top", "right", "bottom" and "left" to adjust your elements position, but how it is positioned depends on what value you set position to:

  • position: static;

    positions based on the content; cannot be adjusted and it's the default value for position for all elements (you probably don't need to know that, but can use it to sound cool at parties)

  • position: relative;

    positions relative to where it normally is

  • position: absolute;

    like "fixed", but positions with respect to the element's nearest relative container

  • position: fixed;

    positions with respect to the browser window (useful for sticky headers and stuff)
